# 错误复盘

本文件记录 AI 在本项目中被用户纠正过的错误，以及以后必须执行的防复发动作。
不要只写“下次认真”，每条都要能指导后续验收。

## 2026-07-05 · 把本地服务断开误说成浏览器拦截

- 场景：用户打开 `three-view-training.html` 时，页面无法访问，我上一轮把问题笼统说成“浏览器安全策略拦截”。
- 我做错了什么：没有第一时间确认 `8089` 静态服务是否还在监听，也没有先用 `curl -I` 区分 `ERR_CONNECTION_REFUSED` 和真正的浏览器自动化限制。
- 用户如何纠正：用户指出不能每次都说被拦截，必须自己解决；用户真正关心的是页面能不能打开、能不能验收。
- 根因：把“自动化工具不能操作错误页”和“用户页面打不开”混在了一起，没有先排查最基础的服务状态。
- 以后必须这样做：遇到本地页面打不开，先跑 `lsof -nP -iTCP:端口 -sTCP:LISTEN` 和 `curl -I 'URL'`。如果是 `ERR_CONNECTION_REFUSED`，先启动服务；只有服务返回 200 后，才进入浏览器自动化验证。
- 可验证的防复发动作：静态训练页统一用 `npm run dev` 启动 `127.0.0.1:8089`；任务记录里必须写清楚服务状态、HTTP 状态和浏览器验证三者分别是否通过。

## 2026-07-05 · 页面验证不等于数学事实验证

- 场景：基础截面页修正正方体/长方体“直角三角形”时，我上一轮说“验证过”，但实际主要验证了按钮、DOM、默认 3D 状态和实时文案。
- 我做错了什么：把页面验收说得像数学事实验收一样完整，没有把“盒体三角截面是否可能被误判为直角三角形”做成单独验证。
- 用户如何纠正：用户指出“你不是说验证过吗”，要求把这类错误写进项目复盘，避免下次再犯。
- 根因：验证层级混淆。UI 状态正确只能证明交互接线和默认状态，不等于证明几何事实、算法边界和拖动过程都正确。
- 以后必须这样做：涉及几何、数学、法律、医学、金融等领域事实时，最终说明必须区分“页面验证 / 算法验证 / 领域事实验证 / 用户体验验证”。
- 可验证的防复发动作：几何结论必须至少有一个确定性样例、抽样扫描、函数测试或推导证据；不能只点击页面默认状态就宣称“事实验证通过”。
- 本次补救：`section-foundation.js` 已让正方体/长方体三角截面分类不再因接近 90 度被误判成“直角三角形”；`tests/reasoning-lesson-layout.test.mjs` 已加入盒体禁用直角误判的源码断言。

## 2026-07-05 · 局部验证不能替代全量截面矩阵

- 场景：基础截面页已经有 3D、实时截面和缩略图，但用户继续发现“长方体三角形实际是五边形”等问题。
- 我做错了什么：上一轮只验证了少量代表按钮和源码断言，没有把所有基础体、所有 can/cannot 卡片组成矩阵逐项验证。
- 用户如何纠正：用户要求“不要让我去验证”，明确指出要先验证旁边小图是否符合标签，再验证右侧 3D 是否呈现对应截面。
- 根因：验收样本太窄。一个页面上有 44 个截面卡片，只测其中几个会让错误预设继续混在页面里。
- 以后必须这样做：涉及基础图形库、题库选项、模板库这类“多卡片/多样例”页面时，必须跑全量矩阵验收；每个卡片都要检查标签、预览、主视图和实时结果是否一致。
- 可验证的防复发动作：新增或调整截面预设时必须运行 `tests/foundation-section-presets.test.mjs`，并在浏览器自动点击所有 44 个基础截面卡片，失败项不得靠肉眼跳过。
- 本次补救：`LESSON-014N` 将 7 个错位预设改为真实交线参数，并新增真实几何矩阵测试；浏览器 44/44 卡片验收通过。

## 2026-07-05 · 约束求解后填充剩余块必须重新校验投影

- 场景：建立第一道三视图黑白块题时，我先用约束方式求出部分方块位置，再把剩余未知位置补成白块。
- 我做错了什么：第一次补白块后没有立刻重新计算左视图/俯视图/主视图，导致左视图多露出两格；幸好 `tests/three-view-training.test.mjs` 在提交前抓到了。
- 用户如何纠正：用户之前已经反复要求“不要让我去验证”，本次按这个要求由专项测试先发现并修正。
- 根因：把“满足部分约束的中间解”误当作“完整题目解”。三视图这类题只要剩余方块位置改变，三个投影都可能改变。
- 以后必须这样做：凡是通过搜索、枚举、AI 生成或手工补全得到的几何题数据，最终必须重新跑完整投影/交线/选项一致性校验，不能只检查数量。
- 可验证的防复发动作：三视图正式题必须运行 `tests/three-view-training.test.mjs`；每道题至少验证块数、颜色数、给定视图、正确选项和模型投影全部一致。
- 本次补救：已替换 `black-white-blocks-001` 的方块坐标；专项测试 5/5 通过，浏览器显示 `validation=pass`。
