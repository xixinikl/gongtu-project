# 公途立体图推与空间几何当前架构

状态：当前有效
版本：2.0
更新日期：2026-07-03

## 1. 产品目标

当前主目标不是通用 CAD，也不是“上传任意图片后自动得到答案”。产品首先服务考公图推中的
立体组合、切面判断和逐项排除，让学生能同时看到题目选项、标准组合模型、真实截面和人工讲解。

已有建模成果不删除。系统保留三个相互独立的入口：

| 产品 | 页面 | 用户 | 核心用途 |
|---|---|---|---|
| 动态解题课 | `/reasoning-lesson.html` | 学生 | 选项验证、关键帧讲解、真实截面和手动探索 |
| 空间几何实验室 | `/geometry.html` | 学生/教研 | 基础体、积木组合、视角、切面与投影自由实验 |
| CSG 工作台 | `/csg-section.html` | 教研/管理员 | 制作、调整和验证组合实体模板 |

三者共享几何模块，但不把复杂建模控件塞进学生解题页。

## 2. 当前技术边界

| 层 | 当前实现 |
|---|---|
| 页面 | 原生 HTML、CSS、JavaScript |
| 3D | Three.js、OrbitControls |
| 组合实体 | `three-bvh-csg`，学生黄金题先生成实体并集再切片 |
| 截面 | Section Engine V2 |
| 题目协议 | `ReasoningCase 1.0-draft` JSON + JSON Schema + 语义校验 |
| 后端与题库 | 现有 FastAPI、SQLite，后续作者工具接入 |
| AI | 可配置的 OpenAI 兼容多模态接口，仅生成待确认草稿 |

真实密钥只能放在 `.env`。AI 输出是不可信输入，不能生成可执行代码，不能决定正式数学答案。

## 3. 共享几何内核

Section Engine V2 是三个产品唯一的生产截面链路：

```text
组合体或三角网格
  → 三角面与无限平面求交
  → 线段端点聚类、去零长、去重复
  → 邻接图链接一个或多个闭环
  → 共享切面二维基
  → 外环、孔洞、洞中岛拓扑
  → Earcut 三角化与面积守恒
  → 稳定复用的填充和轮廓 BufferGeometry
```

旧的“外棱交点 → 质心极角排序”已经废弃：它丢失边连接关系，对凹轮廓和多轮廓不可靠。
Earcut 只负责已验证边界的三角化，不能用来猜轮廓拓扑。

关键模块：

| 模块 | 职责 |
|---|---|
| `triangle-plane-slice.js` | 单三角面输出零或一条规范线段 |
| `section-segment-normalizer.js` | 端点聚类、零长和重复线段清理 |
| `section-contour-builder.js` | 邻接链接闭环，拒绝开链和非流形 |
| `section-contour-topology.js` | 二维投影及外环/孔洞归属 |
| `section-triangulation.js` | 三角化、索引与面积校验 |
| `section-engine-v2.js` | 遍历世界坐标网格并编排完整链路 |
| `section-visual-v2.js` | 稳定显示真实截面并复用 GPU 资源 |

生产默认使用 V2。V1 仅保留为事故定位回退，不得重新成为主算法。

## 4. 动态解题数据与运行链

每道学生题使用一个 `ReasoningCase`，包含：

- 题目来源、题干和选项顺序；
- 参数化组合模型及代表性尺寸；
- 人工编写的几何约束；
- 每项轮廓、可行性、满足/违反的约束和排除理由；
- 相机与切平面关键帧；
- 人工标准答案、验证状态和不确定项。

运行链：

```text
人工黄金题 JSON
  → Schema 与跨字段语义校验
  → 参数模型/CSG 实体
  → Section Engine V2 真实截面
  → 选项状态机
  → 相机与切面时间线
  → 逐项约束讲解
  → 完成后才揭晓人工答案
```

学生页的选项线稿是“待验证候选”，蓝色截面才是几何引擎实时计算的结果。页面不得为了匹配
选项而伪造截面动画。

## 5. 交互职责

- 拖拽 3D 视口：旋转相机视角。
- “手动探索”中的四向按钮或键盘方向键：旋转切平面。
- “切面位置（偏移）”滑条：沿法向量移动切平面。
- 时间线：恢复题目人工定义的相机与切面关键帧。
- 退出手动探索：回到当前讲解关键帧。

相机旋转、切面旋转和切面偏移是三种不同操作，不得再共用一个含糊的拖拽状态。

## 6. 作者与 AI 辅助链

动态解题 MVP 通过后，按以下顺序扩展：

```text
手工讲解编辑器
  → ReasoningCase JSON 导入导出
  → 安全上传与题图/选项裁剪
  → OCR 和轮廓候选
  → 多模态模型生成 Geometry JSON 草稿与 uncertainties
  → Schema 校验 + 几何校验 + 3D 预览
  → 管理员确认
  → 正式题库
```

可使用低成本付费的 OpenAI 兼容接口（包括用户提供的 DeepSeek 配置），也可接入免费开源库。
本地大模型不是 MVP 前置条件。无论供应商是谁，都必须经过同一人工确认门。

## 7. 已验证范围

- Section Engine V2：凸、凹、阶梯、多轮廓、孔洞及连续扫面测试。
- 几何实验室：基础体、组合体、切面、视角和交互状态。
- 动态解题：倒圆锥+方体、四棱锥+圆柱两道人工作品。
- 学生页：三栏响应式布局、选项状态机、时间线、真实 CSG 截面、手动旋转与偏移。

黄金题的尺寸和关键帧是用于教学重建的代表参数；原视频未给出的尺寸不得伪装成题目条件。

## 8. 当前限制

- 单张题图通常不能唯一恢复遮挡结构，必须记录不确定项。
- 相交但未做实体并集的多个壳体可能产生非流形截面；黄金题先做 CSG 并集。
- V1 回退只能用于诊断，不能作为新功能依赖。
- 自动测试不能替代对参考视频的人工体验验收。
- Electron 打包、通用图片录题和正式题库发布仍是后续阶段。

## 9. 近期唯一顺序

1. 完成两道黄金题的逐项约束讲解。
2. 对照两段参考视频完成动态课验收和证据留档。
3. 建立不依赖 AI 的手工讲解编辑器与 JSON 导入导出。
4. 再接入安全上传、OCR、轮廓提取和低成本多模态草稿。

不得跳过黄金题验收直接做“任意图片自动出答案”。
