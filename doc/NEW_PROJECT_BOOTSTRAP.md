# 新项目启动模板

新项目不要先写功能，先建立最小项目记忆。目标是让任何新 Agent 拉下仓库后，能快速知道用户偏好、项目状态、启动方式和完成标准。

## 给新对话框的第一句话

```text
使用 standard-project-workflow。先给这个新项目建立长期 AI 工作区记忆，然后再开始做首个功能。

必须先创建或补齐：
1. AGENTS.md
2. CURRENT_STATUS.md
3. TASKS.md
4. 错误复盘.md
5. doc/USER_PREFERENCES.md
6. doc/ENVIRONMENT.md
7. 一个只读自检命令，例如 npm run doctor / python scripts/doctor.py

要求：
- 不要一次生成很多空文档；
- 每个文件必须写真实项目命令，不要保留模板占位符；
- 用户数据功能默认必须保留登录、登出、数据持久化和账号隔离；
- 完成前必须能说明如何在一台新机器上检查环境并启动。
```

## 新项目最小文件

| 文件 | 作用 |
|---|---|
| `AGENTS.md` | Agent 第一入口，写清先读什么、禁止什么、怎么交接 |
| `CURRENT_STATUS.md` | 当前分支、当前任务、做到哪、下一步 |
| `TASKS.md` | 任务看板、验收标准、完成证据 |
| `错误复盘.md` | 用户纠正过的问题和防复发动作 |
| `doc/USER_PREFERENCES.md` | 用户长期沟通、工程、产品、审美偏好 |
| `doc/ENVIRONMENT.md` | Node/Python/数据库/启动/换机器说明 |
| `doctor` 命令 | 只读检查环境，不安装、不删除、不伪造成功 |

## 新项目第一轮完成标准

第一轮不要追求大功能，先达到：

- 新 Agent 知道先读哪个文件；
- 用户知道项目当前在哪一步；
- 环境问题可以通过一个命令定位；
- 第一个页面、API、脚本或功能可以从真实入口跑通；
- 未完成项和下一步写进 `CURRENT_STATUS.md`。

## 全新 Codex 时

如果换到一个全新的 Codex 环境，不要依赖本机 `~/.codex` 目录。直接把本文件中的“给新对话框的第一句话”发给它，让它在新仓库里创建真实项目记忆。

本机模板只是加速器，不是事实源。真正能跨 Codex 的，是已经写进 Git 仓库的文件。
