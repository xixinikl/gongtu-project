# 考公立体图推教学方法交接

更新时间：2026-07-04

状态：纠偏基线。上一版 `LESSON-013` 只补了文字提示，已被用户明确否决。后续 agent 必须按本文重新理解题目和讲解方式，再动界面。

## 1. 最重要的纠偏

用户要的不是“程序员解释截面定理”，而是“老师带学生看题、贴近选项、逐项排除”的动态过程。

错误方向：

- 只写“这个平面截不出”“违反某规则”，但三维图没有真的演示。
- 把“六边形”本身讲成不可能。
- 候选图是六边形，实际演示却没有做出一个接近六边形的尝试。
- 让用户靠键盘一步步切换，手感不连续。
- 模型过大、缩小时又被下方区域遮住，用户看不全。

正确方向：

- 先看选项像什么，再把切面摆到尽量像这个选项的位置。
- 如果能贴近，就继续检查有没有多出来的边、缺角、曲线、接触转折或被带出的结构。
- 如果不能贴近，要让用户从画面上看见“怎么摆都差一块”，不是只读文字。
- 基础几何知识要先独立讲清，再进入组合体题目。

## 2. 已复核的视频材料

本轮用 `ffmpeg` 对用户给的 8 个本地视频做了约 2 秒间隔的详细抽帧接触表，并逐张检查关键动作。不要声称已逐秒听完整音频；当前依据是视频画面、字幕和关键帧动作。

| 视频 | 讲解重点 | 产品要学到什么 |
|---|---|---|
| 三视图（对比选项区别，排除即可） | 对比选项局部差异，圈出某列/某块不一致 | 三视图训练要支持高亮差异块，不是只给 3D 旋转 |
| 三视图十秒内选答案 | 快速从俯视/左视对应到主视，排除形状差异 | 要做“看两个视图、定位候选差异、旋转验证”的节奏 |
| 选项越简单越容易失误 | 简单选项也要查隐藏边、连接和比例 | 不能因为选项简单就只给结论，必须显示隐含结构 |
| 圆锥+方体经典模型 | 上方方体、下方倒圆锥；A 能贴合，B/D 是相似陷阱 | B 必须做“近似六边形尝试”，再显示会带出方体/圆锥痕迹 |
| 棱锥+圆柱组合截面考察 | 先贴近 A/C/D，再看曲边、缺角、棱锥侧面 | 曲线和直边在组合处必须一起解释 |
| 方体经典题型，选项均是矩形② | 每个矩形候选都要看是不是会多一条横线/断点 | 方块组合题要显示被切到的内部网格线或层级线 |
| 方体经典题型，选项矩形③ | 通过旋转和“拉高/移动”的空间理解排除 | 需要紧凑模型和清楚相机，不然学生看不懂空间关系 |
| 方体经典截面，选项均是矩形 | 切面连续移动，寻找最像候选的矩形，再看左右差异 | 滑动扫截面是核心交互，键盘只能作为备用 |

## 3. 老师的讲解套路

1. 看题面和选项，先找最显眼的形状差异。
2. 选一个候选图，问“如果它是真的，切面大概应该怎么摆”。
3. 在模型上移动或旋转切面，让真实截面尽量贴近候选。
4. 贴近后继续观察：有没有多出来的边、少掉的角、曲直不一致、接触处断开、内部线不该消失。
5. 如果候选是错的，错因要落在画面上：学生应该看到“多出来/带出来/缺掉”的地方。
6. 如果候选是对的，要让候选轮廓和真实截面能在同一视角下对上。

这套流程叫“候选驱动验证”。页面的讲解顺序应该是：

```text
候选图
  -> 摆切面贴近候选
  -> 显示真实截面
  -> 标出多/少/断/曲直/接触差异
  -> 再给一句短解释
```

抽象几何规则只能放在后面，不能替代三维演示。

## 4. 基础截面知识库

这些内容要进基础图形训练入口，不能藏在某道题的长文案里。

| 基础体 | 常见可截形状 | 明确不能直接截出的形状 | 教学备注 |
|---|---|---|---|
| 正方体 | 三角形、正方形、长方形、梯形/平行四边形、五边形、六边形 | 圆、椭圆、任意曲边图形；超过 6 条边的单一凸多边形 | 六边形可行，经典做法是切面同时穿过六个面或相关棱 |
| 长方体 | 三角形、矩形、平行四边形、梯形、五边形、六边形 | 圆、椭圆、曲边；超过 6 条边的单一凸多边形 | 比例会改变，形状不一定正，但边仍是直线 |
| 圆柱 | 水平圆、斜切椭圆、轴向矩形、局部弧边截面 | 纯三角形、纯五边形、纯六边形 | 曲边来自圆柱面，直边来自上下底面或切到侧面边界 |
| 圆锥 | 水平圆、斜切椭圆、过顶点三角形/两条母线、抛物线/双曲线类圆锥曲线 | 纯正方形、纯六边形、无曲线的任意多边形 | 公考题通常看“过顶点直母线”与“不过顶点曲线”的差异 |
| 棱锥 | 三角形、四边形、梯形、五边形等直边多边形 | 圆、椭圆、曲边图形 | 只要切到平面侧面，边界就是直线；斜切常出现缺角 |
| 棱柱/方块组合 | 多边形、L 形、带内部网格线的截面 | 曲边；忽略层级接触的干净图形 | 多个小方块组合时，内部接触线、断点和层级差异很重要 |
| 组合体 | 各部分截痕的真实组合 | 只取其中一个部件的好看截面、忽略另一部分 | 同一无限平面必须同时解释所有被切到的部件 |

参考资料可用于核验基础知识：

- Math is Fun: `https://www.mathsisfun.com/geometry/cross-sections.html`
- MathBitsNotebook: `https://mathbitsnotebook.com/Geometry/3DShapes/3DCrossSections.html`
- Khan Academy: `https://www.khanacademy.org/math/geometry/hs-geo-solids/hs-geo-2d-vs-3d/v/ways-to-cut-a-cube`
- Math is Fun Conic Sections: `https://www.mathsisfun.com/geometry/conic-sections.html`

## 5. 圆锥+方体 B 六边形必须这样返工

当前 `cone-box-001` 的 B 选项是“凸六边形”。正确讲法不是“六边形不可能”，也不是只写“组合体不匹配”。

必须做成一个视觉验证：

1. 候选区显示干净凸六边形。
2. 三维模型上摆一个“尽量接近六边形”的切面。
3. 真实截面区显示实际轮廓，不允许用手绘假图代替 V2 结果。
4. 画面上要能看出：上方方体会留下直边、直杆或转折；下方倒圆锥会留下母线或曲线；接触区会把两者连接起来。
5. 讲解只点一句：“你想要六边形，就必须这样斜切；但这样斜切会把下面/旁边这些结构一起带出来，所以它不是选项里的纯六边形。”

验收标准：

- 用户不读文字，只看三维切面和二维真实截面，也能明白 B 为什么错。
- 真实截面要“像六边形但不等于选项六边形”，不能完全不像。
- 如果当前代表模型无法切出足够接近的画面，先调代表模型比例和关键帧，不要硬讲。

## 6. 交互与画面返工要求

截面验证的手感要像小程序里的连续操作：

- 上下滑动/滚轮：连续移动切面位置，看到一层层截面变化。
- 左右滑动：连续旋转切面角度。
- 方向键：只做桌面备用，不作为主交互。
- 触控和鼠标拖动都要能直接操作，不要求先点“手动探索”。
- 切面、三维橙色截面、二维截面卡片、候选对比必须同步。

画面尺寸要求：

- 模型默认要完整出现在视口内，不能顶满屏幕。
- 切面和真实截面优先可见，允许牺牲一点模型放大比例。
- 缩小模型时不能让它沉到下方卡片后面。
- 应增加自动 fit camera：根据组合体包围盒和切面位置计算相机距离、target 和画面边距。
