# 公途首页中段国风美术与版式方案

更新时间：2026-07-17
对应 Goal：`homepage-content-led-guofeng-redesign` / HCG-2 复核版
上游内容：`doc/HOMEPAGE_MIDDLE_CONTENT.md`

## 设计命题

页面的唯一任务，是让备考用户快速理解：一道题结束后，题目、步骤、追问和错因仍然接得上下一次学习。

视觉母题继续使用 **墨过留痕**：灰蓝墨势代表正在发生的学习过程，暖橙金微光只在墨面局部经过，提示被留下的学习证据。国风来自笔触边缘、雾化层次、字形骨架和留白，不来自人物、建筑、地图或古典器物。

## 本轮纠偏

旧方案把五种能力做成五个同构长章节，导致三个问题：文字太多、颜色相同、每章都像在等待一张配图。现改为：

1. 中段只保留一个核心价值、三个学习时刻和一个“四能力 + 西西教练”入口。
2. 三个学习时刻采用不同背景色与排版方向，不再复制同一模板。
3. 不给每段生成图片。真实功能优先使用真实界面局部，只有整体氛围需要生成素材。
4. 四种学科能力各用一张短入口图，西西教练放在旋转双环中心，不再生成第五张同构图。

| 中明度灰蓝 | `#536C77` | “做完后”，表达沉淀但不压成黑场 |

## 字体与排版

- 主标题：`"Noto Serif SC", serif`，桌面 48–76px，手机 37–43px。
- 正文：`"Noto Serif SC", serif`，桌面 15–16px，手机 14px，行高约 1.9。
- 眉题与状态：`Inter, "PingFang SC", sans-serif`，10–12px。
- 大型背景字只重复当前场景真实关键词，例如“题目 / 作答 / 步骤”，不用“01 / 02 / 03”充当装饰。

## 页面结构

```text
正式首屏（保持不动）
        ↓
深靛展厅｜核心价值：答案结束了，学习不能断
左：短文案        右：宽幅循环氛围
        ↓
展厅中的浅纸面｜做题时：题目和步骤留在一起
        ↓
展厅中的烟青纸面｜追问时：不用再把背景讲一遍
        ↓
展厅中的灰蓝纸面｜做完后：错因回到下一次训练
        ↓
灰蓝雾境｜四张能力主题图 + 中心西西双环
        ↓
深靛展厅｜证据汇聚收束 → 正式尾屏（保持不动）
```

深靛底层从首屏后一直贯穿到尾屏。三个学习场景使用低饱和、近似画布颜料层的线性冷暖色面，不再使用水波、墨团或椭圆雾场。橙金干笔只负责画出标题重点、学习路径与段落显现；桌面交错左右，手机顺序堆叠，每段只保留标题、一句话和一个结果短句。

## 配图职责

### 1. 核心价值：只生成这一张氛围素材

- 比例：宽幅 16:9，主要用于桌面右侧，并保证手机中央裁切。
- 内容：一笔灰蓝墨势经过留白，暖橙金微光沿墨面循环经过，并留下三处痕迹。
- 禁止：人物、建筑、地图、道路、节点图、界面文字、印章和武器。
- 作用：建立参考视频中的冷雾蓝、暖橙微光、笔触边缘和留白气质；不负责解释功能。
- 当前原型不使用上一轮抽象水墨素材，也撤下偏暗、偏写实的桌面静物 v1。新的 `doc/prototypes/assets/gontu-study-record-painted-v2.png` 只画题目纸、步骤草稿、追问页和复盘册；底色为雾白宣纸，使用青绿、湖蓝、杏橙和淡金矿物色，保留细白描、飞白与不完全覆盖的干笔。视频参考只抽取“完整内容画面被白描和颜料逐笔画出”的方法，不抽取水、桥、山景等画面元素。

### 2. 做题时：使用真实界面，不生成图片

- 取真实题目界面的局部，只保留题干、作答状态和一个关键步骤。
- 用干笔遮罩显出界面边缘，让国风笔触包住真实产品，而不是画一张假题目。

### 3. 追问时：使用真实界面，不生成图片

- 取当前题与一组真实问答局部。
- 不生成聊天插画，不复制参考视频人物或构图。

### 4. 做完后：使用粒子反馈，不放章节图

- 前两段的少量证据微粒进入中明度灰蓝背景，最后沉淀为“错因 / 复盘 / 下一步”三个可读关键词。
- 粒子有起点和终点，不做随机飘雪。

### 5. 四能力入口：四图与中心双环

- 言语、数量、图推、申论分别使用一张 4:3 原创主题图；图一只借鉴灰蓝雾境、暖金核心光、白色线描和大颗粒，不复制桥、亭子、山水或构图；图二只借鉴“一项能力一幅主题画”的组织方式，不复制物件。
- 桌面四图同排，手机单列展开；每图只保留能力名称、学习证据和入口。西西教练位于上方双环中心，解释四类学习记录如何被同一个教练承接。
- 双轨道、流动金弧和四个节点复用此前获用户认可的环形语言；Three.js 大颗粒为少量柔焦 Spark 点精灵，悬停轻微增强，`prefers-reduced-motion` 下停止旋转和粒子更新。

### 6. 收束：证据汇聚场

- 继续使用深靛底色，以巨大的“留”字暗纹、缓慢呼吸光场和横向金线完成收束。
- 不再使用普通浅色大标题屏；收束后直接进入正式尾屏。

## 动效边界

| 位置 | 动效 | 目的 |
|---|---|---|
| 核心价值 | 8.5 秒宽幅循环；灰蓝笔触缓慢呼吸、暖橙微光沿墨面经过 | 建立气氛，不抢标题 |
| 段落切换 | 单次 450–650ms 泼墨遮罩 | 在同一灰蓝色相内完成明度过渡 |
| 做题/追问 | 按滚动轻微显影；背景墨晕缓慢漂移；结果短线单次写出 | 证明功能，不制造假内容 |
| 做完后 | 12 个有目标点的粒子沉淀 | 表达学习证据进入复盘 |
| 能力入口 | 四图同排；双环缓慢反向旋转；金弧沿轨道流动；Three/Spark 大粒子缓慢上浮 | 分清四种能力，并表达西西在中心承接记录 |
| 中段收束 | 背景光场呼吸；横向证据线循环写出 | 把前文证据汇聚后无缝交给正式尾屏 |

`prefers-reduced-motion: reduce` 下关闭循环、泼墨、字势雾光、收束光场与粒子，只保留静态内容。

## 桌面与手机要求

- 桌面基准：1280×720；核心价值双栏，三段每段约 520px 高。
- 手机基准：390×844；所有场景改为单列，标题在前、关键词在后，不做左右交错。
- 手机每段约 600–660px，滚动一屏能读完一段并看到下一段颜色。
- 页面不得横向溢出；真实界面局部不得把正文挤成窄列。

## HCG-2 复核结论

当前结构为“三个内容色面 + 一张学习记录手绘图 + 四张能力主题图 + 一组中心双环 + Three/Spark 大粒子”。四图分别对应言语、数量、图推、申论，西西教练在环中心承接四类记录；图一的灰蓝雾境、暖金核心光、白描与大颗粒作为共同风格，不复制其桥景。桌面 1280×720 与手机 390×844 均已实页复核且无横向溢出，当前仍需用户实页确认，不提前视为视觉通过。
