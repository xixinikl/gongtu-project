# 用户长期偏好

本文件记录跨对话、跨 Agent 需要复用的用户偏好。它不是任务看板；具体任务状态看 `CURRENT_STATUS.md` 和 `TASKS.md`。

## 沟通偏好

- 用户是长期搭建自己产品的人，不只想要一次性答案。
- 解释要具体、有步骤，必要时用简单类比，但不要停在概念层。
- 用户说“你看看”通常表示希望 Agent 主动检查上下文和现状。
- 用户说“很乱”通常表示需要先建立秩序、事实源和下一步，不要急着继续堆功能。
- 最终回复要说明做了什么、如何验收、哪些没做、下一步是什么。

## 工程偏好

- 希望 Agent 自动读项目规则和状态，不要每次都靠用户提醒。
- 希望每个项目都能从仓库文件恢复上下文，而不是依赖某个旧聊天。
- 希望依赖、Python 版本、数据库、启动方式都有明确检查命令。
- 希望长期任务有清晰暂停点，额度用完或换 Agent 时不丢上下文。
- 不喜欢“看起来能跑”的临时 demo，偏好能继续维护的纵向切片。

## 产品与数据偏好

- 登录、登出和用户数据保留是默认需求，不需要每次重复提醒。
- 用户数据必须按账号隔离。
- 刷新页面、重启服务、重新登录后，合理的数据应继续存在。
- 本地开发可以使用固定测试账号和开发数据库，但真实用户数据不得入库到 Git。

## UI 审美偏好

- 偏好浅色、柔和、干净、稳定的界面。
- 训练类页面要像正式学习系统，不要像临时实验页。
- 布局要清楚、节奏要稳，按钮和控件要能直接服务学习流程。
- 响应式要实际检查桌面和手机宽度，不能只靠想象。

## Agent 复盘偏好

用户纠正过的错误不要只在聊天里道歉。需要写进 `错误复盘.md`，格式至少包含：

```text
场景：
做错了什么：
根因：
以后必须这样做：
可验证的防复发动作：
```
