# 公途 V3 统一体验修复 Goal

## 权威入口

本文与 `.xds/goals/unified-v3-experience-repair.json` 是本次修复的**唯一权威**进度来源。原始依据是 [gongtu-unified-learning-platform.md](gongtu-unified-learning-platform.md)：`verbal-ai-v3` 已确认的公途黑金、宣纸、古风 SVG 外壳必须落实到真实产品，不能仅保留为 Demo。

## 当前事实

- Phase 6 的技术回归验证了路由、记录、无横向溢出与控制台状态，但没有验证真实页面是否采用确认的 V3 Demo。
- `csg-section.html` 把空间导航放进窄侧栏，依赖没有可发现性的横向滚动；桌面用户无法自然发现或点击完整路径。
- 立体空间、空间学习和三视图正式页面没有面向用户的 AI 教练入口或上下文交接。
- `reasoning.spatial` 的 Skill Registry 与后端上下文存在，但未形成真实用户路径。
- 片段阅读正式页使用独立答题工具外观，与已确认的 V3 外壳有明显断层。

## 目标与范围

将言语、数量、平面图推、立体图推、申论和 AI 教练的**真实正式页面**恢复为可辨认的统一 V3 学习体验：导航清晰可点；从训练带真实且允许的模块/活动/题目/作答或结果/耗时/技能/同源 `return_url` 进入 AI；AI 可安全返回原任务；桌面和手机实际验证后再交由用户确认。

## 非范围

- 不改写题库事实、不伪造 AI 回答或学习记录。
- 不把“无报错、无溢出”当作视觉验收。
- 不自动合并 PR、删除其他工作树或覆盖用户本地修改、数据库与密钥。

### V3R-6 用户确认的最终交互合同（2026-07-14）

以下约束已由用户通过 `?review=inline-tabs-demo` 明确确认，是 V3R-6 后续实现与浏览器验收的直接依据：

1. `/app` 已有的两层导航是产品骨架，不新增第三层外壳，不重做最上方全局模块导航。
2. 言语与数量二级标签必须在原标签下方的宣纸内容区直接切换真实题目、题型或复盘；不得跳转到另一套视觉页面，也不得使用 iframe 嵌套网页。
3. 言语二级标签收口为 `词语积累 / 逻辑填空 / 限时套题 / 我的复盘`；`片段方法` 与重复套题入口合并，进入 `限时套题` 立即出现真实题目。
4. 数量 `今日训练 / 单题诊断 / 套题策略 / 系统学题型 / 我的复盘` 均在原内容区工作；训练标签立即出现真实题目，题型按值得做的顺序展示并能发起同题型训练。
5. “带当前上下文问西西”是题目旁的紧凑辅助区，不占用页面顶部；上下文只能由服务端按当前用户、session/活动重新核验。
6. 右栏只保留当前任务真正有用的进度、顺序、证据或下一步；没有用途的占位内容删除。
7. 图推总览/思维导图页面删除重复的黑色全局栏和白色二级栏；立体工具页删除第一条黑色全局栏，只保留页面自己的工具导航与返回 `/app` 的入口。
8. 申论独立页删除重复注入的统一外壳与二级标签，只保留清晰返回入口；AI 教练最近对话与“新对话”按钮保持可辨认间距。
9. 桌面首屏以无需上下反复翻动为目标；弹出词语卡不得透明、不得溢出、不得在离开/切题后滞留。

上述合同已确认“方向正确”，但真实题库接线、AI 发送、特殊页去壳和最终视觉仍需浏览器证据；因此 V3R-6 继续保持进行中。

## 停止条件

- 发现跨用户数据泄露、伪造事实、错误 Skill、非同源返回 URL 或会覆盖用户改动时，立即停止当前任务并记录。
- 任何页面仍与确认 Demo 有明显断层、导航不可点或 AI 不可到达时，自动测试通过也不能继续宣称完成。
- 未获用户实际看页确认，V3R-6 与整个 Goal 均不得完成。

## 完成审计

完成前逐项检查：六条正式路径均采用 V3 产品层级；导航可见可点；五个业务模块均能把真实允许上下文带入 AI 并安全返回；记录和隔离保持正确；桌面/手机无压缩、遮挡、断层；用户已确认关键页面。任何一项缺失，Goal 继续执行。
