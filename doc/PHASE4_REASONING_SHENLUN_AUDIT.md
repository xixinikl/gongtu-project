# Phase 4 图推与申论事实审计

更新时间：2026-07-13

## 平面图推

- `mindmap.html` 的导图内容保留，错题 CRUD 继续归属 JWT 当前用户。
- 审计发现旧图片可通过 `/data/images/<qid>/...` 枚举读取；正式页面现改用 JWT 图片接口，静态路由阻断旧公开路径。
- 审计发现编辑前端发送 FormData、后端要求 JSON，实际返回 422；现已统一为 JSON 更新。
- 旧复习统计依赖全局 localStorage，换账号会串记录且可能超过 100%；现改为服务端复习会话，支持刷新恢复、忘记回队、重开与 A/B 隔离。
- 创建错题和完成复习会写入 `reasoning.planar` 统一活动；兼容时间线会跳过已经统一索引的旧记录，避免重复计数。

## 立体图推

- 当前正式四段主链为：基础截面、三视图训练、自由切面、组合体切割。
- `reasoning-lesson.html` 保留为题内精讲深链，不作为固定第五导航。
- 新增学习中心和共享四段导航；专业页面保留各自布局，3D 画布未被统一卡片壳替换。
- 三视图组训练由服务端读取正式 50 题题库重新判分；客户端伪造分数会被拒绝。
- 基础、自由切面和组合体只记录访问或明确完成的教学任务，不记录分数或正确率。
- 登录用户的继续学习与历史成绩读取服务端；旧 localStorage 只作为未登录预览，不冒充账号记录。

## 申论

- 内置题库实际只有 10 题：第 1、2 套缺失，第 3 套仅 2 题；现有材料是 103—179 字摘要，不是完整原题材料。
- 部分参考答案和评分点包含摘要未提供的信息，因此全部题目明确标记 `summary-only / isComplete=false`，只能做方法练习，不能称完整真题或客观评分事实。
- 普通提问与正式批改已分开：聊天不进入错题本、不进入弱项证据；只有真实批改才形成历史、问题本和统一学习活动。
- 同题重做保留每次批改历史，问题本只保留当前题的一张聚合卡；单删、清空、重做均按 JWT 当前用户执行。
- 题目列表和详情要求登录；作答前不返回参考答案或评分点。
- AI HTML 一律转义，删除了原样渲染旁路。
- `feiyang-skill` 通过严格 loader 参与批改、对话和弱项分析；每次运行记录 provider、model、skill version 与 SHA-256 hash，缺文件时失败而不是静默降级成空方法论。
- “一般/较差”维度会形成用户可读问题、证据与去重的下一步任务；“优秀/良好”不制造弱项。

## 明确保留的边界

- 未获得申论完整原题材料和可靠来源前，不补造缺题、全文或评分点。
- 立体图推结构与程序校验不能替代学科人工核验；正式题仍遵守已有几何闭环门禁。
- Phase 5 才建设跨模块真实 AI 教练；本阶段只让申论自身的 Skill 与批改真实落地。
