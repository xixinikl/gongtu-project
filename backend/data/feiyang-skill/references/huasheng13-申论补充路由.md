# huasheng13 申论补充路由

## 定位

申论的核心 Skill 仍是“飞扬 · 申论应试思维框架”。本文件只是题型操作补充，不替代飞扬的心智模型、决策启发式和公文模板。

## 来源与选择

- 上游：`WangJunqing-coder/huasheng13-skill`，固定 commit `6a43d776741f69a231eb2d75f9d5d59efe870659`。
- 默认只参考 `references/shenlun.md` 的“审题—找点—加工—书写”流程。
- 只有语言规范问题才参考 `references/shenlun-biaoda.md` 的口语转规范表达与常见表达错误。
- 不默认加载 `references/shenlun-sucai.md`、时政热点、真题示例、练习集或整套范文；素材必须来自当前题目材料或用户明确提供的可信来源。

## 按题型补充

| 题型 | 补充检查 |
| --- | --- |
| 归纳概括 | 问什么答什么；按材料找点；同类合并；控制概括层级 |
| 综合分析 | 明确分析对象与题干要求；按材料建立观点—依据—结论 |
| 提出对策 | 对策必须对应材料中的问题/原因，并检查主体、动作和可执行性 |
| 公文写作 | 先确定身份、对象、目的和文种，再调用飞扬公文模板核对格式 |
| 大作文 | 给立意、结构、分论点和材料使用方向，不生成可直接抄写的整篇范文 |

## 冲突与诚实边界

当前材料、题干要求和服务端记录是事实源。若补充方法与飞扬核心规则冲突，优先采用更具体、可在当前材料中验证且符合题干要求的规则，并明确说明依据；不得把上游整理内容说成官方评分标准。
