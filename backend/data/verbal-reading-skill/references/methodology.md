## 1. 通用诊断链

每题按固定顺序检查，避免只看答案字母：

1. **问法**：主旨、意图、标题、细节、填入、排序、上下文推断还是词句理解；先识别正向/反向设问。
2. **结构**：总分、分总、转折、因果、问题—对策、观点—论证、并列、时间对比或句群衔接。
3. **主体/主题**：谁或什么是全文持续讨论的对象；答案是否丢主体、换主体或扩大范围。
4. **重点句**：观点、对策、结论、定义、转折后、尾句新内容；例子和背景通常是论证，不自动等于主旨。
5. **选项比较**：正确项应覆盖重点且边界恰当；逐项检查片面、无中生有、范围扩大、偷换概念、绝对化、非对策、背景项和问题项。

诊断要指出偏离发生在哪一步。例如：

- 找到了尾句，但选项比较时丢掉“农民”主体；
- 识别了问题，却把问题项当成作者最终对策；
- 逐项定位正确，但忽略反向设问；
- 只看局部例子，没有覆盖并列的另一层。

## 2. 题型方法

### 中心理解与意图判断

- 先找主体，再找观点或对策。
- 后对策结构优先概括对策，但必须保留对策对象和条件。
- 并列结构选择能覆盖各层的全面概括，警惕只概括一层的片面项。
- 观点—论证结构中，例子、数据和原因服务于观点。

### 标题拟定/填入

- 标题同时检查主体、范围和吸引力，准确性优先于文艺性。
- 排除缺主体、过宽、过细、只写例子或只写背景的标题。

### 细节判断

- 看清“正确/不正确/没有提及”等设问方向。
- 对每个选项定位原文，检查对象、程度、时态、因果和条件。
- “原文出现过相似词”不等于选项正确。

### 语句填入

- 横线在中间时分别概括前后文，判断承上、启下或承上启下。
- 横线在结尾时判断总结、对策、结论或话题延续。
- 保持主体、话题、指代和逻辑词一致。

### 语句排序

- 先分句群，再找首句候选、指代词、关联词、时间顺序、定义—解释和总分关系。
- 用首尾呼应和句群衔接验证完整顺序，不只凭一个首句猜答案。

### 下文/上文推断

- 下文不重复已讲完内容，优先承接尾句新问题、新概念或待展开对象。
- 上文应能自然引出首句，保持话题和逻辑方向一致。

### 词句理解

- 把词句放回上下文，优先解释其在本文中的功能和指代。
- 排除字面扩大、脱离语境和替换对象的选项。

## 3. 高频错误模式

| 错误模式 | 识别证据 | 诊断表达 |
| --- | --- | --- |
| 审错问法 | 反向设问却选符合项 | 问法识别失败，不是材料没读懂 |
| 主体丢失/偷换 | 正确项保留主体，错误项换成更大或更小概念 | 找到重点但选项落地时丢主体 |
| 问题项冒充主旨 | 文段末尾有对策，用户选择只描述问题 | 没完成从问题到对策的转折 |
| 片面概括 | 文段并列两层，选项只覆盖一层 | 分层正确但缺全面概括 |
| 无中生有 | 选项加入原文没有的比较、目的、结果 | 被合理常识吸引，越过文本证据 |
| 范围扩大 | 从具体主体扩大为整个领域 | 选项看似正确但边界过宽 |
| 绝对化 | 原文“可能/部分”，选项“必然/全部” | 忽略程度词 |
| 背景/例子当重点 | 选择开头背景或醒目例子 | 没区分观点与论证材料 |
| 衔接单边 | 填入项只接前文或只启后文 | 未同时验证两侧逻辑 |
| 排序局部猜测 | 只靠首句，不验证句群 | 缺少完整衔接校验 |

不要把正确选项标签当作弱项。`正确概括`、`正确排序`、`正确对策`等只能解释答案，不进入错误模式统计。

## 4. 证据强度与置信度

- 同一错误模式出现3题及以上：可称“本套主要弱项”，置信度通常可到0.75以上。
- 出现2题：可称“重复风险”，结合题型和用时调整置信度。
- 只出现1题：称“本套暴露的风险”，置信度不宜高于0.6。
- 全对或只有未作答：不能编造选项陷阱弱项；优先分析速度、未作答和复盘质量。
- 历史趋势只在输入明确包含多次会话时使用。

置信度不是数学精确概率，而是证据充分程度。说明依据，不营造虚假精确。

## 5. 训练建议

训练建议必须单点、可执行、可复测：

- 先选一个主要弱项；
- 推荐3—5道匹配题；
- 说明匹配的题型、结构、主体或陷阱标签；
- 做题时要求记录一个具体动作，例如“圈主体”“写出对策对象”“逐项定位程度词”；
- 训练后用同一标准复测，不只看答案是否正确。
